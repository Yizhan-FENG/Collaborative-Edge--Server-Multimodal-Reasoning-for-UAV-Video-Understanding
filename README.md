# UAV-EdgeServer-Multimodal-Reasoning

This repository provides the implementation of an edge--server collaborative multimodal reasoning framework for UAV video understanding. The framework converts raw UAV videos into compact structured multimodal evidence, including semantic keyframes, BLIP-2 captions, and YOLO-based visual logs, and then performs server-side multimodal reasoning using Qwen3-VL.

## Models

The models used in the experiments are listed below:

- `blip2-opt-6.7b`
- `yoloe-v8l-seg.pt`
- `Qwen3-VL-8B-Instruct`

Please download the required models by yourself and store them in appropriate local directories. The model paths should be manually modified in the corresponding scripts before running the code.

**Note:** All file paths in the code should be modified according to your actual local environment.

## Data

The `data` directory contains the processed states of the MSR-VTT and MSVD-QA datasets, as well as the WordNet resource used by the NLTK library.

Please note that this repository does not include the original videos or annotation files of MSR-VTT and MSVD-QA.

https://drive.google.com/drive/folders/1A8IYY6jw_zfc5QAGVFk1gj8AlwY4KNBt?usp=sharing

For each dataset folder, the main contents are:

* `keyframe/`: selected video keyframes generated by the edge-side experiment.
* `structured_reports/`: structured BLIP-2 captions and YOLO detection logs generated from the raw edge-side outputs.
* `videos_analysis_results.jsonl`: raw records produced by the edge-side pipeline.

## Edge-Side Experiment

The `edge_experiment` directory contains the edge-side processing pipeline, which is used to select semantic keyframes, generate BLIP-2 semantic captions, and produce YOLO-based detection logs.

Main scripts:

* `edge_BLIP2_videos.py`
  Runs the edge-side video analysis pipeline. After execution, the selected keyframes and the raw output file `videos_analysis_results.jsonl` will be generated in the corresponding dataset folder under `data`.

* `process_edge_output.py`
  Processes the raw edge-side output records and generates the `structured_reports/` folder.

## Center-Side Experiment

The `center_experiment` directory contains the server-side experiments, including comparative experiments and ablation experiments.

### Ablation Experiment

The `Ablation_experiment` directory contains four ablation settings:

* `full_model`
* `only_keyframe`
* `without_summary_information`
* `without_yolo_information`

Each setting includes two tasks:

* `description`: video description generation using Qwen3-VL-8B-Instruct.
* `QA`: video question answering using Qwen3-VL-8B-Instruct.

For the video description task, the Python scripts perform server-side inference, and the `qwen3_vl_results/` folder stores the metadata JSONL files and generated result JSONL files.

For the QA task, the Python scripts perform server-side question answering. The `qwen3_vl_results/` folder stores the metadata and model output files, while the `qa_comparison_results/` folder stores the extracted key-answer JSONL files for subsequent quantitative evaluation.

### Comparative Experiment

The `Comparative_experiment` directory contains the comparative settings:

* `full_model`: the full proposed pipeline, consistent with the full model setting in the ablation experiment.
* `only_Qwen3`: direct processing of raw videos using Qwen3-VL-8B-Instruct.

## Evaluation

The `val` directory contains the evaluation scripts for both video description and QA tasks.

### Video Description Evaluation

The `description` folder contains:

* `BERTScore.py`: computes BERTScore F1 between generated descriptions and reference captions.
* `four.py`: computes BLEU, METEOR, ROUGE-L, and CIDEr.

### Question Answering Evaluation

The `QA` folder contains:

* `BERTScore.py`: computes BERTScore F1 between generated answers and ground-truth answers.
* `LLM_test.py`: evaluates semantic similarity between generated answers and ground-truth answers using Qwen3-8B-Instruct.
* `wordnet.py`: evaluates answer similarity using WordNet-based matching.

## Single-Video Inference Example

The `main_path_single` directory provides an example for single-video inference. It can be used to test the complete pipeline on one video sample.
